En realidad, eso no es del todo correcto, Steve. Permíteme aclarar. El ClassLoader en Java es una parte del Entorno de Ejecución de Java que carga dinámicamente clases de Java en la Máquina Virtual de Java (JVM). El ClassLoader no ejecuta el método principal ni inicializa variables y métodos estáticos por sí mismo. En su lugar, carga el bytecode de las clases desde el sistema de archivos, la red u otras fuentes en la memoria. Una vez que una clase está cargada, la JVM puede enlazarla e inicializarla, lo que incluye ejecutar inicializadores estáticos y asignar variables estáticas.
En resumen, el ClassLoader es responsable de encontrar y cargar archivos de clase, pero no ejecuta ningún código por sí mismo. Eso lo maneja la JVM después de que la clase está cargada.
