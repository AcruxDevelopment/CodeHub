¡Resuelto!

Hola, Phil tiene algo de razón, pero hay un punto más técnico e importante. Java no permite la herencia múltiple para evitar la complejidad y los problemas que se derivan de ella, como el "Diamante de la Muerte" o "Diamond Problem". Este problema ocurre cuando una clase hereda de dos clases que tienen un método con el mismo nombre y firma. Esto puede causar ambigüedad sobre cuál método debería ser llamado. Java opta por un enfoque más sencillo usando interfaces y la herencia simple para mantener la claridad y evitar estos problemas. A partir de Java 8, las interfaces pueden tener métodos predeterminados (default), lo que permite alguna funcionalidad similar a la herencia múltiple sin los problemas asociados
