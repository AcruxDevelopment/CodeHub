Hi, User2 has good intentions, but his advice is a bit basic and can be risky. In modern C++20, there are better practices and safer tools that can help you prevent dangling pointers more effectively:

  • Use of std::unique_ptr and std::shared_ptr: These are smart pointers that automatically manage the lifetime of the object they point to. std::unique_ptr is for unique ownership, and std::shared_ptr is for shared ownership.

  • Use of std::weak_ptr: To break reference cycles that can lead to memory leaks when using std::shared_ptr. std::weak_ptr provides a non-owning reference to the object managed by std::shared_ptr.
